# DuctZip Roadmap

更新时间：2026-07-02

## 路线图原则

这份路线图结合了竞品调研和 `OAI_WhitePaper_Codex-maxxing26.pdf` 中关于长期工作的建议。DuctZip 应把开发过程沉淀为可审查的 Markdown 文档，让调研、决策、状态和待办可以在后续线程中继续使用。

白皮书对本项目最有用的实践：

- 使用长期线程承载项目上下文。
- 把关键记忆写入仓库文档，而不是只留在聊天记录中。
- 为每个阶段设置可验证目标。
- 把路线图拆成小循环，每轮完成后更新状态。
- 对不可逆动作保留人工确认。

- 单顶层目录不重复包一层。
- 目标冲突时有清晰选择。
