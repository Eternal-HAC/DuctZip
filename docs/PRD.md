# DuctZip PRD

更新时间：2026-07-02

## 1. 产品定位

DuctZip 是一个面向 Windows 的轻量解压工具。第一阶段目标不是替代 Windows 文件资源管理器，也不是重新实现压缩算法，而是提供一个可靠、清晰、适合日常使用的解压流程。

核心定位：

- 以解压为核心，不以压缩为第一阶段目标。
- 优先调用成熟的 7-Zip 后端，不自研解压算法。
- 先完成 CLI 和解压核心，再进入 GUI。
- 默认重视中文路径、错误提示、路径安全和可维护结构。

## 2. 目标用户

### 主要用户

- Windows 日常用户：经常下载 ZIP、7z、RAR 等压缩包，希望快速解压。
- 个人开发者或重度文件用户：需要稳定处理中文路径、空格路径和批量文件。
- 对 7-Zip 功能不熟悉、但希望有更清晰默认解压体验的用户。

### 非目标用户

- 需要完整文件管理器替代 Explorer 的用户。
- 需要复杂压缩参数、分卷压缩、自解压包生成的高级压缩用户。
- 需要云盘聚合、跨设备同步、文件索引系统的用户。

## 3. 问题定义

Windows 上常见解压体验存在以下问题：

- 用户需要手动选择解压目录，容易把多个文件散落到当前目录。
- CLI 工具强大但错误提示不友好。
- 中文路径、空格路径、密码压缩包、损坏压缩包容易暴露细节错误。
- 一些工具功能很多，但第一步操作不够直接。
- 压缩包存在路径穿越和覆盖风险，需要默认防护。

DuctZip 第一阶段要解决的是：让解压流程稳定、可测试、可扩展，而不是堆叠所有功能。

## 4. v0.1 CLI 原型

### 目标

证明 DuctZip 可以稳定发现并调用 7-Zip，完成基础命令行解压。

### 用户故事

- 作为用户，我可以提供一个压缩包路径和输出目录，让 DuctZip 解压到指定位置。
- 作为用户，我可以解压包含中文路径或空格路径的压缩包。
- 作为用户，当 7-Zip 不存在、压缩包不存在或解压失败时，我能看到清晰错误。

### 功能范围

必须支持：

- 发现本机 7-Zip 后端。
- 支持用户显式指定 `7z.exe` 路径。
- 接收压缩包路径。
- 接收输出目录。
- 调用 7-Zip CLI 执行解压。
- 支持 ZIP。
- 支持 7z。
- 支持 RAR 提取，前提是本机 7-Zip 后端支持。
- 支持中文路径和空格路径。
- 输出成功或失败结果。
- 将常见 7-Zip 错误转换为 DuctZip 错误类型。

暂不支持：

- GUI。
- 右键菜单。
- 压缩功能。
- 批量解压。
- Smart Extraction。
- 解压前浏览压缩包内容。
- 随软件内置 7-Zip 二进制。

### 输入输出

CLI 最小输入：

```text
ductzip extract <archive_path> --output <output_dir>
```

可选输入：

```text
ductzip extract <archive_path> --output <output_dir> --sevenzip <path_to_7z.exe>
```

预期输出：

- 成功：显示解压完成和输出目录。
- 失败：显示可读错误信息和退出码。

错误信息不应直接把 7-Zip 原始输出完整抛给普通用户。原始输出可进入调试日志。

### v0.1 验收标准

- 可以解压一个普通 ZIP 文件。
- 可以解压一个普通 7z 文件。
- 可以提取一个 RAR 文件，若后端支持 RAR。
- 可以处理中文压缩包路径、中文输出目录、路径中包含空格的情况。
- 7-Zip 缺失时提示“未找到 7-Zip 后端”。
- 7-Zip 缺失时提示用户安装 7-Zip 或使用 `--sevenzip` 指定路径。
- 压缩包不存在时提示“找不到压缩包”。
- 输出目录不存在时可以创建，或给出明确错误。第一版默认创建目录。
- 解压失败时程序返回非零退出码。

## 5. v0.2 解压核心

### 目标

把 v0.1 的 CLI 调用升级为可复用的解压核心，为后续 GUI、批量任务和右键菜单做准备。

### 用户故事

- 作为用户，我可以在解压前看到压缩包内容。
- 作为用户，我可以测试压缩包是否损坏。
- 作为用户，遇到密码压缩包时能输入密码。
- 作为用户，我能看到解压进度和当前文件。
- 作为用户，我可以取消正在进行的任务。
- 作为用户，我不希望恶意压缩包写出到危险路径。

### 功能范围

必须支持：

- `list_archive()`：列出压缩包内容。
- `test_archive()`：测试压缩包完整性。
- `extract_archive()`：执行解压并返回进度事件。
- 密码需求识别。
- 密码错误识别。
- 任务取消。
- 进度事件。
- 覆盖策略。
- 路径穿越防护。
- 错误类型归一化。

暂不支持：

- 图形界面。
- Windows 右键菜单。
- 批量队列。
- 自动更新。
- 文件预览。
- 云盘和网络位置。

### 核心对象

`ArchiveEngine`：

- 封装后端能力。
- 第一版实现为 `SevenZipCliEngine`。
- UI 和 CLI 只能通过该接口解压，不直接调用 `7z.exe`。

`ExtractRequest`：

- `archive_path`
- `output_dir`
- `password`
- `overwrite_policy`
- `smart_extract`
- `open_after_done`

v0.2 中 `smart_extract` 和 `open_after_done` 可以先保留字段，但不要求完整实现。

`ProgressEvent`：

- `kind`
- `percent`
- `current_file`
- `message`

### 错误模型

v0.2 至少需要定义并处理：

- `SevenZipMissing`
- `ArchiveNotFound`
- `UnsupportedFormat`
- `PasswordRequired`
- `WrongPassword`
- `CorruptedArchive`
- `PathTraversalBlocked`
- `OutputPermissionDenied`
- `UnknownArchiveError`

### 安全要求

必须默认启用：

- 阻止 `..` 路径穿越。
- 阻止绝对路径写出到用户指定目录之外。
- 覆盖已有文件前应用统一覆盖策略。
- 密码不写入日志。

v0.2 暂不强制实现 Mark-of-the-Web 传播，但设计上不得阻碍后续加入。

### v0.2 验收标准

- 可以列出 ZIP/7z 压缩包内容。
- 可以测试正常压缩包并返回成功。
- 可以识别损坏压缩包并返回 `CorruptedArchive`。
- 可以识别密码需求。
- 密码错误返回 `WrongPassword`。
- 解压时能产生至少开始、进度、完成、失败四类事件。
- 用户取消任务后后端进程被终止，任务状态为取消。
- 恶意路径压缩包被阻止，不会写到输出目录之外。
- Core 层和 UI 无耦合，后续 GUI 可直接复用。

## 6. 用户流程

### v0.1 CLI 流程

```text
用户输入命令
-> DuctZip 校验 archive_path
-> DuctZip 准备 output_dir
-> DuctZip 查找 7-Zip
-> DuctZip 调用 7-Zip 解压
-> DuctZip 映射结果
-> 输出成功或错误
```

### v0.2 核心流程

```text
收到 ExtractRequest
-> 校验路径和输出目录
-> 检查压缩包内容
-> 应用安全策略
-> 启动 7-Zip 后端
-> 解析进度事件
-> 处理取消/错误/完成
-> 返回结构化结果
```

## 7. 非功能要求

可靠性：

- 不因路径中包含中文或空格失败。
- 后端失败时返回可读错误，而不是崩溃。

可维护性：

- UI、CLI、ArchiveEngine、Core 分层清楚。
- 7-Zip 调用细节只存在于 ArchiveEngine 实现中。

安全性：

- 默认阻止危险路径。
- 不记录密码。
- 错误日志避免泄露大量用户文件名。

可测试性：

- 解压核心可以在没有 GUI 的情况下测试。
- 错误映射可以用固定 7-Zip 输出样例测试。

## 8. 明确不做

v0.1/v0.2 不做：

- 压缩功能。
- 图形界面。
- Windows 右键菜单。
- 批量解压。
- Smart Extraction 完整实现。
- 内置 7-Zip 二进制。
- 完整文件管理器。
- 云盘管理。
- 自动更新。
- 自研压缩算法。
- fork NanaZip 或 7-Zip。

## 9. 后续衔接

v0.1/v0.2 完成后进入：

- v0.3：PySide6 GUI 原型。
- v0.4：Smart Extraction。
- v0.5：批量解压。
- v0.6：Windows 右键菜单。

这些阶段不得绕过 v0.2 的核心接口直接调用 7-Zip。
